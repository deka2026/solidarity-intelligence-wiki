# 온톨로지 — 지식을 분류하고 연결하는 뼈대

> **한 줄 요약**: 온톨로지는 "이 세계에 무엇이 있고, 그것들이 어떻게 관계를 맺는가"를 정리하는 틀이다

**태그**: `#온톨로지` `#지식관리` `#분류체계` `#리뷰어역량`  
**작성자**: 데카 (김일영)  
**작성일**: 2026-06-27

---

## 1. 온톨로지란 무엇인가

### 철학에서의 온톨로지

온톨로지(Ontology)는 원래 철학 용어로, **"존재하는 것에 대한 탐구"**입니다. 그리스어 *ontos*(존재) + *logos*(학문)에서 왔습니다.

- "세상에는 어떤 종류의 것들이 있는가?"
- "그것들은 서로 어떤 관계인가?"
- "어떤 것이 다른 것보다 더 근본적인가?"

이런 질문을 다루는 분야입니다.

### 정보과학에서의 온톨로지

컴퓨터 과학과 지식관리에서는 같은 단어를 더 실용적으로 씁니다:

> **온톨로지 = 특정 영역의 개념들과 그 관계를 명시적으로 정의한 것**

쉽게 말하면, **"이 분야에서 쓰는 단어들의 사전 + 그 단어들 사이의 관계 지도"**입니다.

### 일상의 비유

도서관을 생각해보세요:

| 도서관 | 온톨로지 |
|--------|---------|
| 분류 기호 (000, 100, 200...) | **개념의 범주** |
| 같은 칸에 꽂힌 책들 | **같은 범주에 속한 것들** |
| "이 책도 참고하세요" 안내 | **개념 간 관계** |
| 도서관 사서의 판단 | **온톨로지 설계자(= 리뷰어)의 역할** |

사서가 없는 도서관에서는 책을 찾을 수 없듯, 온톨로지가 없는 위키에서는 지식을 찾을 수 없습니다.

## 2. 온톨로지의 구성 요소

온톨로지는 4가지 요소로 이루어집니다:

### (1) 클래스 (Class) — "어떤 종류가 있는가"

우리 위키에서의 클래스:

```
지식
├── 개념 (concept)
├── 현장기록 (field-note)
├── 실습가이드 (how-to)
├── 사례연구 (case-study)
└── 참고자료 (resource)
```

이 분류 자체가 온톨로지입니다. "위키에 올라오는 지식은 이 5가지 종류가 있다"고 선언한 것이니까요.

### (2) 인스턴스 (Instance) — "구체적으로 어떤 것이 있는가"

클래스가 "종류"라면 인스턴스는 "실물"입니다:

| 클래스 | 인스턴스 (예시) |
|--------|---------------|
| 개념 | "연대지능", "AI 프롬프트", "마을자치" |
| 현장기록 | "0627 전주 마을회의 참관" |
| 사례연구 | "망님마을 인스타 콘텐츠 제작" |

### (3) 속성 (Property) — "각 것에 대해 무엇을 알아야 하는가"

우리 템플릿의 상단 메타데이터가 바로 속성입니다:

```
태그: ...        ← 이 문서의 주제어
작성자: ...      ← 누가 썼는가  
작성일: ...      ← 언제 썼는가
장소: ...        ← 어디서의 기록인가 (현장기록만)
난이도: ...      ← 얼마나 어려운가 (실습가이드만)
```

속성이 일관되어야 문서를 검색하고 비교할 수 있습니다.

### (4) 관계 (Relation) — "것들 사이에 어떤 연결이 있는가"

가장 중요한 부분입니다. 지식은 고립되면 죽습니다.

```
[연대지능] ──── 의 사례이다 ──── [망님마을 콘텐츠 제작]
     │                              │
     │                              │
  상위 개념이다                  에서 사용된다
     │                              │
     ▼                              ▼
[집단지성]                     [AI 프롬프트 작성법]
```

우리 위키에서 `[[링크]]`로 연결하는 것이 바로 관계를 만드는 행위입니다.

## 3. 왜 리뷰어에게 온톨로지가 중요한가

### 리뷰어 = 위키의 사서

활동가는 자기 경험을 씁니다. 리뷰어는 그 글이 **전체 지식 체계 안에서 올바른 자리를 찾도록** 돕는 사람입니다.

| 리뷰어가 하는 일 | 온톨로지 관점 |
|----------------|-------------|
| "이건 concept가 아니라 how-to에요" | **클래스 분류 교정** |
| "태그에 #마을자치를 추가해주세요" | **속성 보완** |
| "이미 비슷한 문서가 있으니 링크를 걸어주세요" | **관계 연결** |
| "이 용어는 우리 위키에서 '연대지능'으로 통일합니다" | **용어 표준화** |
| "이 주제는 두 문서로 나누는 게 좋겠어요" | **클래스 세분화** |

### 온톨로지 없이 리뷰하면 생기는 문제

```
❌ 같은 개념을 "집단지성", "공동지능", "협력지성"으로 다르게 부름
   → 검색해도 안 나옴, 연결이 끊어짐

❌ "연대지능 개념 + 활용 사례 + 사용법"이 한 문서에 다 들어감
   → 너무 길고, 부분 수정이 어려움

❌ 관련 문서끼리 링크가 없음
   → 섬처럼 떠 있는 지식, 맥락 없이 읽어야 함

❌ 현장기록에 장소가 빠져 있음
   → 나중에 "전주 관련 기록"을 모아볼 수 없음
```

## 4. 실전: 리뷰할 때 묻는 5가지 질문

PR이 들어왔을 때, 아래 질문을 순서대로 던져보세요:

### Q1. 이것은 무엇인가? (클래스 확인)

> "이 글은 개념 정리인가, 현장 기록인가, 사용법인가, 사례 분석인가?"

올바른 폴더에 있는지 확인합니다. 경계가 애매할 때는 **주된 목적**으로 판단합니다.

- 마을회의에서 AI를 써본 이야기 → 경험이 중심이면 **현장기록**, 방법이 중심이면 **실습가이드**

### Q2. 무엇이라고 부르는가? (용어 확인)

> "이 글에서 쓰는 용어가 우리 위키의 기존 용어와 일치하는가?"

위키 전체에서 같은 것을 같은 이름으로 부르는 것이 중요합니다.

- "AI 비서" vs "AI 도구" vs "AI 어시스턴트" → 하나로 통일

### Q3. 무엇을 알려주는가? (속성 확인)

> "템플릿의 필수 항목(태그, 요약, 작성자, 날짜 등)이 빠짐없이 채워져 있는가?"

빈 칸이 있으면 나중에 검색과 분류가 안 됩니다.

### Q4. 무엇과 연결되는가? (관계 확인)

> "기존 문서 중 이 글과 관련된 것이 있는가? `[[링크]]`가 걸려 있는가?"

이것이 온톨로지에서 **가장 가치 있는** 리뷰 행위입니다. 활동가는 자기 글만 보지만, 리뷰어는 전체를 봅니다.

### Q5. 이것이 이미 있는가? (중복 확인)

> "같은 주제의 문서가 이미 있지 않은가?"

있다면 세 가지 선택지:
1. 기존 문서에 **병합**하도록 안내
2. 기존 문서와 **차별점**을 명확히 하도록 요청
3. 기존 문서를 **링크**로 참조하도록 요청

## 5. 우리 위키의 온톨로지 설계

현재 우리 위키의 온톨로지를 그림으로 보면:

```
                    ┌─────────────────┐
                    │   연대지능 위키   │
                    └────────┬────────┘
                             │
            ┌────────┬───────┼───────┬──────────┐
            ▼        ▼       ▼       ▼          ▼
        ┌──────┐ ┌──────┐ ┌────┐ ┌──────┐ ┌────────┐
        │개념  │ │현장  │ │실습│ │사례  │ │참고    │
        │정리  │ │기록  │ │가이│ │연구  │ │자료    │
        │      │ │      │ │드  │ │      │ │        │
        └──┬───┘ └──┬───┘ └─┬──┘ └──┬───┘ └────────┘
           │        │       │       │
  속성:    │        │       │       │
  - 태그   │        │       │       │
  - 요약   │   + 장소│  + 난이│  + 지역│
  - 작성자 │   + 활동│  + 소요│  + 결과│
  - 작성일 │     일  │   시간 │       │
           │        │       │       │
           └────────┴───┬───┴───────┘
                        │
                    [[링크]]로
                     서로 연결
```

### 앞으로 성장할 수 있는 방향

위키가 커지면 온톨로지도 함께 자랍니다:

| 지금 | 나중에 추가될 수 있는 것 |
|------|----------------------|
| 4개 문서 유형 | 인터뷰 기록, 정책 분석, 도구 리뷰 등 새 유형 |
| 태그 자유 입력 | 표준 태그 목록 (통제어휘) |
| 단순 링크 | "상위개념-하위개념", "원인-결과" 등 관계 유형 |
| 문서 단위 | 문서 안의 개별 주장(claim) 단위 검증 |

이 성장은 한번에 설계하는 게 아니라, 문서가 쌓이면서 리뷰어가 **패턴을 발견하고 구조를 다듬는** 과정입니다.
