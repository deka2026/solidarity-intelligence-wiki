# 단일 HTML 사이트에 회원 등급(무료·유료·관리자) 붙이기

> **한 줄 요약**: 서버 없이 단일 HTML 파일 하나로 일반회원(무료)·정회원(유료)·관리자를 나누고 메뉴 접근을 제어하는 방법

**태그**: `#웹사이트` `#햇빛소득마을` `#회원제` `#ClaudeCode` `#입문`
**작성자**: 데카 (김일영)
**작성일**: 2026-07-24
**난이도**: 보통
**소요 시간**: 약 30분 (읽고 따라하기)

---

## 이 가이드가 필요한 상황

마을·협동조합 사이트를 만들 때 "이건 누구나 보게, 저건 회원만, 관리 기능은 나만" 처럼 나누고 싶을 때. 예: 전남광주햇빛소득마을네트워크 사이트는 소개·수익추정·가입문의는 누구나, 서류관리·경영공시·질문답변은 정회원만, 게시물 삭제·회원 관리는 관리자만 쓰게 했습니다.

## 핵심 아이디어

별도 서버·DB 없이 **브라우저 저장소(localStorage)** 만으로 로그인·등급을 흉내 낼 수 있습니다. 데모·검토 단계에 충분하고, 나중에 Supabase 같은 실제 로그인으로 바꿔 끼울 수 있습니다.

등급은 4개로 나눴습니다.
- **일반회원(무료)**: 로그인 없이 이용
- **정회원(유료)**: 로그인 필요, 실무 기능 사용
- **메뉴관리자 / 총관리자**: 답변·삭제·회원 관리

## 단계별 진행

### 1단계: 메뉴마다 "필요 등급"을 표시

각 메뉴 버튼에 "이건 어떤 등급이 필요한지"를 속성으로 적어둡니다.

```html
<div class="tab" data-sec="filing" data-need="paid">📁 서류관리</div>
<div class="tab" data-sec="admin" data-need="admin">🛠 관리자</div>
```

`data-need`가 없으면 무료, `paid`면 정회원, `admin`이면 관리자 전용이라는 뜻입니다.

### 2단계: 메뉴를 누를 때 자격을 확인

메뉴 전환 함수에서, 필요한 등급이 없으면 로그인 창을 띄우거나 안내합니다. 비회원이 유료 메뉴를 누르면 자동으로 로그인 창이 열리게 하면 자연스럽습니다.

### 3단계: 관리자가 등급을 올려줌

신규 가입자는 자동으로 무료 회원이 됩니다. 회비를 낸 회원을 총관리자가 "회원 관리" 화면에서 정회원으로 바꿔주는 흐름입니다. 이렇게 하면 유료 결제 시스템 없이도 운영 초기를 넘길 수 있습니다.

### 4단계: 데모 계정을 미리 넣어두기

검토하는 사람이 바로 눌러볼 수 있게 샘플 계정을 심어둡니다. (예: `admin`/`manager`/`haetbit`, 비밀번호 `sun2026`)

## 주의할 점

- localStorage는 **그 브라우저에만** 저장됩니다. 다른 기기에서 보면 데이터가 없습니다 → 실서비스 전엔 Supabase 등으로 교체.
- 비밀번호를 화면 코드에 그대로 두는 방식은 **데모 전용**입니다. 실제 운영에선 진짜 로그인(암호화 저장)을 써야 합니다.
- 여러 사이트를 합칠 때는 저장소 이름(접두어)을 달리해 데이터가 섞이지 않게 합니다.
