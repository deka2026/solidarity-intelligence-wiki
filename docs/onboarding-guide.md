# 연대지능 활동가 온보딩 가이드

GitHub를 처음 쓰는 분도 따라할 수 있도록 한 단계씩 안내합니다.

---

## 전체 흐름

```
[1] GitHub 가입 → [2] 중앙 위키 Fork → [3] 내 위키에서 글쓰기 → [4] PR 제출 → [5] 검수·피드백 → [6] 머지!
```

---

## 1단계: GitHub 가입

1. https://github.com 에 접속합니다
2. **Sign up** 클릭
3. 이메일, 비밀번호, 사용자 이름을 입력합니다
   - 사용자 이름 추천: `이름-지역` (예: `jiyoung-jeonju`, `minho-gangwon`)
4. 이메일 인증을 완료합니다

---

## 2단계: 중앙 위키 Fork (내 복사본 만들기)

> Fork = 원본 저장소를 내 계정으로 복사하는 것. 원본에 영향을 주지 않고 자유롭게 작업할 수 있습니다.

1. 중앙 위키 저장소에 접속합니다:
   `https://github.com/deka2026/solidarity-intelligence-wiki`
2. 오른쪽 상단의 **Fork** 버튼을 클릭합니다
3. "Create fork" 클릭
4. 잠시 후 내 계정에 `내아이디/solidarity-intelligence-wiki`가 생깁니다

---

## 3단계: 문서 작성하기

### 방법 A: GitHub 웹에서 직접 (가장 쉬움)

1. 내 Fork 저장소에 들어갑니다
2. 글을 쓸 폴더로 이동합니다:
   - 개념 정리 → `wiki/concepts/`
   - 현장 기록 → `wiki/field-notes/`
   - 실습 가이드 → `wiki/how-to/`
   - 사례 연구 → `wiki/case-studies/`
3. **Add file** → **Create new file** 클릭
4. 파일 이름을 입력합니다 (예: `ai-chatbot-basics.md`)
   - 반드시 `.md`로 끝나야 합니다
   - 소문자, 하이픈(-) 사용 (예: `마을-회의-진행법.md`)
5. 템플릿을 복사해서 붙여넣고 내용을 채웁니다
   - 템플릿은 `templates/` 폴더에 있습니다
6. 아래쪽 **Commit changes** 클릭

### 방법 B: Claude Code 활용 (추천)

Claude Code에서 아래처럼 요청하면 템플릿에 맞춰 문서를 생성해줍니다:

```
"마을 주민 인터뷰 현장 기록을 field-note 템플릿으로 작성해줘"
```

---

## 4단계: Pull Request 제출하기 (중앙 위키에 기여)

> PR = "내가 쓴 글을 중앙 위키에 올려주세요"라고 요청하는 것

1. 내 Fork 저장소에서 **Contribute** → **Open pull request** 클릭
2. 제목을 씁니다:
   - 형식: `[유형] 문서 제목` 
   - 예: `[concept] AI 챗봇 기초 개념 정리`
   - 예: `[field-note] 0627 전주 마을회의 참관 기록`
3. 설명란에 아래 내용을 채웁니다:

```markdown
## 어떤 문서인가요?
(한두 줄로 요약)

## 작성 배경
(왜 이 문서를 쓰게 되었는지)

## 셀프 체크
- [ ] 템플릿 구조를 따랐습니다
- [ ] 태그를 3개 이상 달았습니다
- [ ] 관련 문서 링크를 확인했습니다
```

4. **Create pull request** 클릭
5. 리뷰어가 배정되면 피드백을 기다립니다

---

## 5단계: 검수 피드백 받기

리뷰어가 PR에 코멘트를 남기면:

1. **이메일 알림** 또는 GitHub 알림으로 확인합니다
2. 코멘트 내용을 읽고, 수정이 필요한 부분을 고칩니다
3. 내 Fork에서 해당 파일을 수정하면 **PR에 자동 반영**됩니다
4. "수정했습니다"라고 코멘트를 남기면 리뷰어가 재확인합니다

### 자주 받는 피드백과 대응

| 피드백 | 의미 | 할 일 |
|--------|------|-------|
| "출처를 추가해주세요" | 근거가 필요합니다 | 참고한 URL이나 자료명 추가 |
| "좀 더 구체적으로" | 예시가 부족합니다 | 실제 사례나 수치 추가 |
| "기존 문서와 중복" | 이미 비슷한 글이 있습니다 | 기존 문서에 병합하는 방식으로 수정 |
| "LGTM" | Looks Good To Me | 승인! 곧 머지됩니다 |

---

## 6단계: 머지 (축하합니다!)

리뷰어가 승인하면 관리자가 PR을 **머지**합니다.
내가 쓴 문서가 중앙 위키의 공식 지식이 됩니다.

---

## 내 Fork를 최신 상태로 유지하기

다른 활동가의 글이 중앙 위키에 머지되면, 내 Fork에는 자동 반영되지 않습니다.
주기적으로 동기화해주세요:

1. 내 Fork 저장소에서 **Sync fork** 버튼 클릭
2. **Update branch** 클릭
3. 끝!

## 도움이 필요할 때

- **글쓰기**: Claude Code에게 "이 내용을 위키 템플릿으로 정리해줘"라고 요청하세요
- **검수 관련**: PR 코멘트에 질문을 남기면 리뷰어가 답변합니다
- **기술 문제**: 사교원 슬랙 `#위키-도움` 채널에 질문하세요
