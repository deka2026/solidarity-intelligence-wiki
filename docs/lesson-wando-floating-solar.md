# 레슨 — 완도군 수상태양광 사업계획서 작업 (2026-07-15)

## 1. 재사용할 스크립트는 세션 스크래치패드에 두면 사라진다

- 스크래치패드는 **세션별 폴더**라 다음 세션에서 접근 불가. 7/15 오전 세션에서 검증한 build.js/extract.js를 오후 세션에서 처음부터 재작성했다.
- **교훈:** 검증된 도구 스크립트는 그 자리에서 위키 저장소에 전문을 커밋할 것. 이번에 `docs/skill-hwpx-generate-from-template.md`에 전문 수록으로 해결.

## 2. 사업계획서는 폴더 안의 '공고 원문'부터 읽어라

- 사용자 지시(수상태양광·1MW 이하·행정리 협동조합·기본소득 자부담·지방소멸대응기금)만으로 쓰지 않고, 같은 폴더의 **공모 공고 PDF를 pdftotext로 먼저 전문 추출**했더니 문서의 구체성이 크게 올라감:
  - 990kW 규모 설계 근거(300kW~1,000kW 요건 + 500kW 이상 REC 가중치), 개인 출자 10% 상한, 주민 70% 동의 + 총회 의결, 선정 후 6개월 내 발전사업허가, 2차 마감 7.31 같은 실제 요건·기한이 전부 공고에서 나옴
  - 공고가 "농업 기반 자원(저수지 등) 부지 발굴 지원"과 "지방소멸대응기금 연계"를 명시하고 있어 사용자 모델의 제도적 근거를 그대로 인용할 수 있었음
- **교훈:** 관공서 제출물은 창작 전에 근거 공고·지침의 요건 수치를 문서에 정렬시키는 것이 품질의 절반이다.

## 3. 확인 안 되는 고유명사는 지어내지 않는다

- 완도군의 실제 저수지 이름을 특정할 수 없어 "군내 대규모 농업용 저수지 등 2~3개소(농어촌공사 관리, 타당성 조사로 확정)"로 처리. 읍면 구성(3읍 9면)·섬 265개처럼 확실한 사실만 명시.
- 재무 수치(구획당 17억, 연 매출 2억 등)는 "개략치(안)"임을 문서 안에 밝힘.

## 4. 셸 체이닝 함정

- Git Bash에서 `node -e "..." && powershell -Command "..."`를 한 줄로 묶으면 PowerShell이 앞의 JS 코드를 받아 파싱 오류를 낸다(인용 경계 붕괴). **PowerShell 검증은 별도 도구 호출로 분리**할 것.

## 5. node zlib.crc32 내장 확인

- Node v24(이 PC)는 `zlib.crc32()`가 내장이라 예전처럼 CRC 테이블을 손으로 만들 필요 없음. zip 직접 쓰기가 훨씬 짧아졌다.
