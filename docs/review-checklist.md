# 검수 체크리스트

Pull Request 리뷰 시 아래 항목을 확인합니다.
리뷰어는 각 항목에 대해 통과(v), 수정필요(x), 해당없음(-)을 표시합니다.

---

## 1. 형식 검수

- [ ] **템플릿 준수**: 해당 유형의 템플릿 구조를 따르고 있는가
- [ ] **제목**: 명확하고 검색 가능한 제목인가
- [ ] **태그**: 관련 태그가 3~5개 포함되어 있는가
- [ ] **한 줄 요약**: 문서 상단에 요약이 있는가
- [ ] **파일 위치**: 올바른 폴더(`concepts/`, `field-notes/` 등)에 있는가
- [ ] **파일명**: `소문자-하이픈-연결.md` 형식인가

## 2. 내용 검수

- [ ] **정확성**: 주장에 근거(출처, 경험, 데이터)가 있는가
- [ ] **완결성**: 배경 지식 없이도 이해할 수 있는가
- [ ] **구체성**: 추상적 서술 대신 구체적 예시가 있는가
- [ ] **분량**: 너무 짧지(200자 미만) 않고, 너무 길지(원고지 20매 초과) 않은가

## 3. 연결 검수

- [ ] **내부 링크**: 관련된 기존 문서에 `[[링크]]`가 걸려 있는가
- [ ] **외부 링크**: 참고 URL이 유효한가 (깨진 링크 없는가)
- [ ] **중복 확인**: 이미 같은 주제의 문서가 있지 않은가 (있으면 병합 제안)

## 4. 품질 검수 (선택)

- [ ] **읽는 맛**: 문장이 자연스럽고 읽기 쉬운가
- [ ] **시각 자료**: 필요한 곳에 표, 그림, 다이어그램이 있는가
- [ ] **실행 가능성**: how-to 문서의 경우, 따라하면 실제로 되는가

---

## 리뷰 결과 표기

PR 코멘트에 아래 형식으로 남깁니다:

```
## 검수 결과

- 형식: v (통과)
- 내용: x (수정 필요) → "3번 단락에 출처를 추가해주세요"
- 연결: v (통과)
- 품질: - (해당없음)

**종합**: 수정 후 승인
```

## 리뷰어 배정 기준

| 문서 유형 | 1차 리뷰어 | 2차 리뷰어 |
|----------|-----------|-----------|
| concepts | 사교원 직원 | 해당 분야 선배 활동가 |
| field-notes | 같은 지역 활동가 | 사교원 직원 |
| how-to | 해당 도구 숙련자 | 사교원 직원 |
| case-studies | 사교원 직원 | 관련 지역 활동가 |
