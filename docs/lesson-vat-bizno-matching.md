# 레슨: 회계 자료 대조 작업에서 배운 것 (부가세 신고 준비)

**작성일:** 2026-07-15

## 1. 대조 키는 사업자등록번호, 그러나 번호도 완전하지 않다

- 상호명은 파일마다 표기가 달라(전각 괄호, 지사명, 결제대행 가맹점명) 매칭 키로 못 쓴다. 번호가 기본.
- 하지만 **같은 실질 거래처라도 번호가 다를 수 있다**: 세금계산서는 본사 번호, 보조금카드 집행내역은 결제대행(KICC 등) 가맹 번호로 찍히는 사례 확인(SK인텔릭스: 104-86-48203 vs 116-81-19948). 번호 매칭 결과가 0이라고 "거래 없음"이라 단정하지 말고, 거래처명 눈대중 교차확인을 한 번 더 할 것.

## 2. 집계 숫자 간 차이는 그 자체가 정보다

- 홈택스 발급건수 > 보조금 집행건수(예: 롯데렌탈 12 vs 7)이면 차액분이 다른 예산·자부담·다른 사업 집행일 가능성. 신고 때 안분 근거가 된다.
- 세액이 비어 있는 매입 행은 면세 계산서일 수 있으므로 부가세 공제 대상에서 따로 분류해야 한다.

## 3. 민감 자료와 공개 위키의 경계

- 회계 원본(건별 금액·집행 용도)은 공개 저장소에 올리지 않는다. 핸드오버에는 **요약(거래처·건수)과 로컬 파일 경로만** 남기고, 상세는 D: 드라이브 CSV로.

## 4. 작업 환경 제약을 먼저 확인하면 시간을 아낀다

- xlsx 스킬(openpyxl 전제)을 그대로 따라가기 전에 `python --version` 한 번으로 스토어 스텁임을 확인 → 바로 Excel COM 경로로 전환해 헛수고를 줄였다. 이 PC 도구 지도는 [[skill-excel-com-no-python]]에 정리.
