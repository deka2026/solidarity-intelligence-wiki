# 핸드오버 — 시민기업펀드 세부사업계획서 작성 + 한글(hwpx) 변환

> 작성일: 2026-07-24 (작업일: 2026-07-22~24) | 작성: 데카(김일영) + Claude Code
> 브랜치: `claude/citizen-enterprise-fund-plan-rrsilc`

## 한 줄 요약

「시민주권 공동체 자산 형성 사업」의 세부사업인 **시민기업펀드**(= 시민기업육성재단의 투자·자본 부문) 계획서를, 사용자가 준 **시민기금 계획서(PDF)의 서식·논리에 맞춰** 작성하고, **템플릿 없이 hwpx로 생성**해 전달했다. 사용자가 한글에서 열어 편집·재업로드했고(→ 생성 hwpx의 한글 열림 확인됨), 그 수정본을 저장소 마크다운에 역동기화했다.

## 사업 모델의 핵심 (도메인 개념 — 다음 세션 필독)

- **상위사업**: 시민주권 공동체 자산 형성 사업(총 2,210억). 원칙은 '민간재단 설립'.
- **세 그릇의 역할 분담**(시민기금 계획서 7장 역할분담표 기준):
  - **시민기금** = 재생에너지 자부담(15%) **융자**(빌려주고 상환받음, 회수형).
  - **시민기업펀드** = 시민기업 **지분 자본**(주주로 소유·배당·상장수익 회수). 본 계획서.
  - 그 외 신용보증재단(보증)·재생에너지투자공사(정책금융)·상호금융(대출)·시민자산지원센터(컨설팅).
- **위상**: 시민기업펀드는 독립 재단이 아니라 **시민기업육성재단(민간재단)의 투자·자본 부문**. 재단이 설립준비→창업→성장→상장지수펀드→연계 플랫폼을 운영, 펀드가 각 단계에 자본을 대고 회수·재순환.
- **★ 용어 구분(사용자가 명시적으로 정정)** — 혼동 금지:
  - **공동체이익회사(CIC)** = 현장의 **개별** 공동체 기업(마을발전소 협동조합 등).
  - **시민기업** = **사회연대경제조직의 한 유형**. ① 공동체이익회사의 **연합조직**이거나 ② 공동체이익회사에 **사업·서비스를 제공**하며 **시민 투자로 운영**되는 기업. → **펀드의 주된 투자 대상**.
  - 둘은 같은 것이 아니라 **층위가 다르다**. 문서에 두 용어가 함께 나오는 것은 혼재가 아니라 구분이다.
- **재원(5+1)**: ① 통합시 출연금(마중물·매칭·후순위) ② 시민펀딩 ③ 공동체이익회사 출자금(연대출자 회전) ④ 적립금 ⑤ **기초지자체 지역기업 한정 기본소득 일부 적립금**.
- **자산잠금 요건**: 사용자 수정으로 "청산·매각 시 **시민기업펀드 투자액의 우선 상환**"(투자자 보호형). 부록 CIC 정의(공동체 귀속형)와는 성격이 다르니, 향후 일관화 시 주의.

## 작업 경로 (반복 지시 이력)

1. 초안: 시민기금·시민구단 참조 요청 → 저장소엔 원문 없어 완도 핸드오버의 생태계 맥락으로 작성
2. 정정①: "시민구단 아니라 **시민기업재단**" → 위상 재구성
3. 정정②: 정식 명칭 "**시민기업육성재단**"
4. 정정③: 시민기업펀드 = 재단의 **투자부문**(AskUserQuestion으로 확인)
5. 참조: **시민기금 계획서 PDF** 제공 → pymupdf로 전문 추출, 그 서식·법리로 전면 재작성
6. 변환: **hwpx 생성**(템플릿 없음) → 구조검증 통과 → 전달 → **사용자 한글 편집·재업로드**
7. 동기화: 수정본을 마크다운에 반영 + 시민기업/공동체이익회사 관계 정정

## 남은 작업 / 판단 필요

1. **번호 체계**: 한글본에서 '위상'과 '추진배경'이 둘 다 "1"이던 중복을, '위상'을 **번호 없는 도입절**로 두어 해소(본문 1~9장·내부참조 "3장" 유지). 위상을 정식 1장으로 올릴지는 사용자 확인 대기.
2. **hwpx 재생성 여부**: 마크다운은 최신이지만, 사용자가 한글에서 추가한 서식은 마크다운에 없다. 텍스트 기준 재생성이 필요할 때만 진행.
3. **용어 일관화**: 자산잠금 본문(우선상환) ↔ 부록 CIC 정의(공동체 귀속) 성격 차이. 통일 방향은 사용자 지정 대기.
4. 규모·예산·매칭 비율은 전부 **표준가정 기반 추정치**임을 문서에 명시. 시민기금 확정 수치 받으면 교체.
