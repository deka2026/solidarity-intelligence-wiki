## 문제와 해결 — 한글에서 표가 왼쪽으로 collapse

- 증상: 한글(한컴오피스)에서 열자 모든 표·해설상자가 폭을 잃고 왼쪽에 얇은 세로선으로 뭉쳐 내용이 안 보임. (Word에서는 정상)
- 원인: 표 너비를 **백분율(`WidthType.PERCENTAGE`, 100%)** 로 지정. Word는 백분율을 해석하지만 **한글은 절대 너비를 요구**해 폭 계산이 0에 가깝게 무너짐.
- 해결(`generate.js`):
  1. 본문 유효폭 `CONTENT_W = 8.27in - 좌우여백(1300*2) ≈ 9309 twips`를 계산
  2. 모든 표·상자에 `width: {size: CONTENT_W, type: DXA}` + `columnWidths`(각 열 twips) + `layout: TableLayoutType.FIXED`
  3. 셀에도 `width DXA` 명시, 셀 내부 `\n` 줄바꿈 지원, 숫자 정렬코드(0/1/2)→`AlignmentType` 매퍼 추가
- 검증: 재생성 후 `word/document.xml`에서 `<w:tblW ... w:type="dxa">` 120개 / `pct` 0개 / `gridCol` 정상 확인.

## 스킬

### 1. 긴 한국어 문서(docx)는 "블록 DSL + 렌더러" 2파일로 분리 생성
- 내용(`content.js`: `[{t:"h1",...},{t:"table",...}]` 배열)과 서식(`generate.js`) 분리 → 내용 수정·분량 조절이 쉽고, 서식은 한 번만 다듬으면 전체 일관.
- 블록 타입: `h1/h2/h3, p/lead, key/tip/warn/quote/recap(색상 상자), bullets/steps, table, spacer/divider/pb`. 인라인 `**굵게**`/`__강조__` 파서.
- 색상 상자는 **1셀 표(single-cell table)에 배경색+왼쪽 굵은 테두리**로 구현(콜아웃 박스). 문단 테두리(w:pBdr)는 한글에서 깨지므로 쓰지 않음([[handover-hwpx-line-fix]]의 레슨 반영).

### 2. 한글(Hancom) 호환 docx 표의 철칙
- **표 너비는 절대값(dxa) + `columnWidths` + `TableLayoutType.FIXED`**. 백분율(%)은 한글에서 collapse된다.
- 셀 너비도 dxa로 명시. 열 폭 합계 = 본문 유효폭이 되도록 마지막 열에서 보정.
- 정렬은 `AlignmentType`(문자열 enum). 숫자로 넣으면 무시되니 매퍼로 변환.

### 3. 이 PC(파이썬 스텁)에서 docx 생성
- `python` 실행 불가 → **node + `npm install docx`**(인터넷 되면 설치됨, 이번엔 8.5.0)로 생성. python-docx 계열은 못 씀.
- A4 세로높이 = `convertInchesToTwip(11.69)`, 폰트 맑은 고딕, 쪽번호는 footer에 `PageNumber.CURRENT`.

### 4. 렌더러 없이 페이지 수 추정
- 블록마다 세로 twips(줄수×줄높이 + 여백, 표는 행수×행높이)를 누적, 페이지 유효높이 넘으면 +1쪽. `h1`은 페이지나눔이므로 새 쪽 강제.
- 파라미터(줄당 글자수 CPL, 줄높이)를 낙관/보수 2세트로 돌려 브래킷으로 보고 → 단정 대신 범위로 안전하게.

### 5. hwpx 원문 텍스트 추출
- hwpx는 zip → `Contents/section0.xml`의 `<hp:t>` 텍스트를 sed로 이어붙여 원문 복원(파이썬 없이). `Preview/PrvText.txt`는 앞부분만 있어 불충분.

## 레슨

1. **"50쪽 분량"은 렌더링으로 검증하려 하되, 도구가 없으면 추정 모델을 만들어 범위로 보고하라.** 실제 페이지를 셀 수 없다고 감으로 넘기지 말 것. 추정치가 목표에 미달하면(초기 38쪽) 채워야 할 양을 역산해 **의미 있는 콘텐츠**(워크드 예시·수치 요약표·FAQ·실행 체크리스트)로 늘린다 — 빈 여백/반복이 아니라.
2. **원문 "목차 유지"는 제목을 글자 그대로 두는 것.** 해설집에서 추가한 부분(프롤로그·기초편·부록)은 원문 목차와 시각적으로 구분(kicker "원문 01" vs "부록 A")해 원문 훼손처럼 보이지 않게.
3. **워드에서 잘 나와도 한글에서 깨질 수 있다 — 대상 뷰어로 확인.** 이번엔 사용자 캡처(한글) 덕에 표 collapse를 잡았다. 한글로 볼 문서는 표 너비 dxa, 문단 테두리 금지가 기본.
4. **산출물과 함께 "재생성 수단"을 남겨라.** 스크래치패드는 세션 종료 시 사라지므로, generate.js+content.js+README를 사용자 폴더에 복사해 두면 이후 사용자가 직접 내용 수정·재생성 가능.
5. **비용·제도·자격 수치가 든 정책 문서는 반드시 "참고치·최신 확인 필요" 고지를 문서 안에 박아라.** 원문의 유의사항을 해설집에도 승계.
