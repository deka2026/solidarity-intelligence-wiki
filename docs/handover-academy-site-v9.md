## 스킬

### 1. 단일 파일 SPA에서 메뉴 통합 반복 패턴
여러 하위 페이지를 한 페이지로 합칠 때: (1) 사이드바 링크를 1개로, (2) 각 섹션을 h3 소제목+초록 밑줄로 구분, (3) 옛 page div는 sed로 뒤에서부터 삭제, (4) 렌더 함수가 참조하는 DOM id를 그대로 옮기면 JS 수정 최소화, (5) go() 디스패치와 userPages/adminPages 배열 갱신. 교육과정·연대지능·활동가의견·성과·교육일정공지 5곳에 동일 적용.

### 2. 정적 사이트 localStorage → Supabase 승격 패턴
기존 getX/saveX(localStorage)에 sbSaveX() 한 줄만 추가하고, sbSync()에서 로드. sbReady 플래그로 테이블 미존재/네트워크 오류 시 자동 폴백. 테이블 생성 SQL은 사용자가 대시보드에서 실행(권한 분리).

### 3. 출처 기반 문항 생성 — 반드시 원문 확인 후
검증 게임 문항 확장 시 raw.githubusercontent.com에서 위키 문서 원문을 먼저 읽고, 문서에 실제로 있는 사실만으로 문항 작성. 추측 금지. good/error/improve 판정도 원문 대조로 결정.

### 2. 자기모순 문항 주의 (검증 게임 품질)
"80% 단축 = 5배"는 수학적으로 참인데 'improve'로 판정한 문항이 학습자에게 혼란. 정답/오답이 원문 근거로 명확히 갈리는 문항만 두고, 추론이 필요한 애매한 문항은 교체.

### 3. 팀 구독과 API는 별개 (사용자 안내)
claude.ai 팀 구독으로는 API를 못 씀. API는 console.anthropic.com에서 종량제(구독 아님). Haiku 4.5 "구독 신청"은 개념이 없고 코드에서 모델 지정 + 크레딧 결제 방식. 결제·키 발급은 사용자 직접(대행 불가).

### 4. 중복 ID 방지 — 페이지 통합 시 필수 확인
위키 sync 카드를 성과 페이지로 옮길 때 관리 페이지의 동일 id(syncPrData 등)와 충돌. getElementById는 첫 매치만 반환하므로 옛 카드 제거 후 grep -c로 id 중복 1개인지 반드시 확인.
