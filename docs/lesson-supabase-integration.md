# 레슨: 기존 localStorage 앱에 Supabase 백엔드 붙이기

**작성일:** 2026-06-29

## 핵심 교훈

단일 HTML + localStorage 앱에 Supabase를 추가할 때, **한 번에 전부 바꾸지 말고 계층별로 진행**하는 게 안전하다.

### 단계별 접근 (이번에 실제 적용한 순서)

1. **데이터 동기화부터** — localStorage를 캐시로 유지하면서 Supabase를 source of truth로 추가. 기존 코드 변경 최소화.
2. **읽기/쓰기 함수 래핑** — 기존 `saveFeedbacks()` 같은 함수 안에 Supabase 저장 한 줄 추가. 기존 로직 그대로 동작.
3. **인증은 마지막에** — 데이터 동기화가 검증된 후 인증 전환. 인증은 가장 많은 부분에 영향을 미치므로 단독 단계로 분리.

### 왜 이 순서인가

- 데이터 동기화는 **기존 코드를 거의 안 건드림** → 안전
- 인증 전환은 **가입/로그인/로그아웃/세션관리/권한체크** 모두 영향 → 리스크 큼
- 인증에서 외부 서비스 의존 문제(이메일 전송, 트리거 충돌)가 발생하면 데이터는 이미 안전

### 하이브리드 패턴의 장점

```
Supabase (source of truth) ←→ localStorage (캐시/오프라인)
```

- 네트워크 없어도 앱 동작 (localStorage 캐시)
- Supabase 장애 시에도 기존 기능 유지
- 다른 기기에서 접속하면 Supabase에서 최신 데이터 로드

### 주의할 점

- ID 충돌: JS에서 생성한 ID와 DB의 auto-increment가 충돌할 수 있음
- RLS: 개발 중에는 열어두고, 프로덕션 전에 조여야 함
- 트리거: Supabase Auth와 DB 트리거 조합은 예상치 못한 충돌 발생 가능
