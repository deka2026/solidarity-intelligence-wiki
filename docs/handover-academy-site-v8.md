## 현재 메뉴 구조

- **홈**: 연대지능네트워크(첫 화면) / 활동가 의견(의견 게시판 + 검증 종합 리포트 통합)
- **교육과정**: AI 학습 가이드(11단계) / 현재 상태 & 다음 단계(진단)
- **연대지능**: 연대지능 학습 가이드(7단계) / 연대지능 위키
- **검증 게임**: 지식 검증 챌린지 / 내 검증 기록
- **성과**: 위키 활동 평가 / 성과 나누기 / 나의 학습 기록 / 비밀번호 변경
- **관리**(관리자만): 품질 리뷰 대시보드 / 교육생 관리 / 교육 일정 / 공지사항

## 핵심 구조 (다음 작업자를 위한 지도)

- 전체 사이트는 `index.html` 단일 파일 (약 4,500줄, 인라인 CSS+JS)
- **AI 학습 가이드**: `LEARN_STEPS` 배열(11단계) + `sendLearnChat()` — 키워드 기반 응답
- **연대지능 학습 가이드**: `SOLID_STEPS` 배열(7단계) + `sendSolidChat()` — 동일 패턴
- **검증 게임**: `GAME_STATEMENTS`(300문제) + `GAME_POOL_MONTH`('2026-07') + `GAME_SESSION_SIZE`(5)
  - 이번 달 안 푼 문제 우선 출제, 진행률 표시(`renderGameMonthProgress`)
- **홈 AI 대화**: `KB` 배열 키워드 매칭 — 실제 Claude API 연결 아님(향후 과제)
- 학습 진행률: localStorage에 이메일별 저장 (`academy_learn_progress`, `academy_solid_progress`)

## 스킬 (이번 작업에서 확보한 기술)

### 1. 단일 파일 SPA에서 대화형 학습 시스템 패턴

키워드 기반이라도 단계(steps) 배열 + 진행률 localStorage + 3버튼(시작/저장/재개) 조합이면
서버 없이 대화형 학습 UX를 만들 수 있다. 동일 패턴을 교육과정(11단계)과 연대지능(7단계)에 재사용했다.

```
STEPS 배열 → showStep(idx) → sendChat()에서 키워드 분기(완료/도움/기타) → 진행률 저장
```

### 2. 월별 문제 풀 + 미출제 우선 로직

localStorage 검증 기록에서 이번 달(`date`가 `YYYY-MM`으로 시작) 푼 문제를 집계해
안 푼 문제를 우선 출제하면, 고정 문제은행으로도 "매달 300문제" 체계를 만들 수 있다.

### 3. CSS 한 곳 수정으로 전 페이지 반영

의견 입력란이 모든 페이지에서 작게 나오는 문제는 `.fb-inline textarea`에
`width:100%; box-sizing:border-box; display:block`만 추가해 일괄 해결.
반복 요소는 인라인 스타일이 아니라 클래스 CSS로 관리해야 수정 비용이 낮다.

## 레슨 (주의사항·교훈)

### 1. 셔플 함수가 사실 고정 순서였다

`(i*7+3)%(i+1)` 같은 의사 셔플은 매번 같은 순서를 만든다. 검증 게임이 늘 같은 문제를
내고 있었던 원인. 셔플은 반드시 `Math.random()` 기반 Fisher-Yates로 작성할 것.

### 2. "새 문제를 만들라"는 요청 전에 현황 확인

문제를 새로 만들라는 요청을 받았을 때 확인해보니 이미 300문제가 위키 문서 기반으로
완성되어 있었다. 코드를 먼저 읽고 사실을 보고한 뒤 필요한 부분(출제 방식)만 고치는 것이
맞다. 추측으로 전면 재작성했다면 멀쩡한 자산을 날릴 뻔했다.

### 3. 페이지 통합 시 렌더 함수의 DOM id 의존 확인

검증 리포트를 의견 게시판 페이지로 합칠 때 `renderVerifyReport()`가 참조하는
`vr-*` id 요소들을 그대로 옮기면 JS 수정 없이 통합된다. 단, 옛 페이지를 지운 뒤
`grep`으로 id 중복(1개만 남았는지)을 반드시 확인할 것.

### 4. sed 줄 번호 삭제는 뒤에서부터

여러 구간을 줄 번호로 지울 때는 뒤 구간부터 지워야 앞 구간 번호가 밀리지 않는다.
