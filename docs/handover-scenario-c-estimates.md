## 한 줄 요약

업로드된 3개 문서(MOU 제안서 초안, 전력수급 시나리오, 시민주권 공동체 자산 형성 사업)를 근거로, ① 연구용역 인력·예산 산출내역서(8개월, 약 1.74억), ② 시나리오 C 분산형 투자(2.0조)의 경제유발·일자리 효과, ③ 시민기업 4대 분야 시장규모와 CIC 순이익 10% 환류액을 연쇄적으로 추정하고, 최종적으로 **산식·기준지표를 전부 공개한 A4 5쪽 상세판 PDF**로 통합했다.

## 배경 · 과제 흐름 (요청 순서대로)

1. MOU 제안서의 연구용역(확산 매뉴얼 제작) → 인력 구성 + 산출내역서 (8개월) → 2쪽 워드 → PDF 출력
2. 전력수급 시나리오 문서의 **시나리오 C**로 실행 시, 분산형 재생에너지(태양광+마이크로그리드) 2027~4년 투자 대비 가정용·농어촌 전기화의 경제유발·일자리 효과
3. 시민주권 문서와 연결해 시민기업(재생에너지·MG·전기화·사회서비스) 시장규모 + 공동체이익회사(발전수익 10% 투자) 환류액 — **기준을 순이익으로 수정**(사용자 지시)
4. ②+③ 통합 인쇄본(3쪽) → 산식·기준지표·논리전개 보강 상세판(5쪽)

## 핵심 수치 체계 (추정 간 정합성)

- 물량 축: 마을발전소 700kW × 1,400개소 = 980MW → 980MW × 8,760h × 15% ≈ **1,300GWh/년** (시나리오 문서와 정합)
- 돈 축: 투자 **1조 9,660억**(PV 12억/MW + ESS 5억/MWh + MG 개소당 2억) / 발전매출 **연 1,950억**(150원/kWh) / 순이익률 **45%**(매출100 − O&M11 − 관리4 − 감가상각25 − 금융8 − 세금7)
- 계수: 생산유발 1.9, 부가가치 0.70, 취업유발 9.0명/10억 (한국은행 산업연관표 부문계수의 투자비중 55/35/10 가중평균)
- **환류 기준은 순이익**: 매출 기준으로 정의하면 10년 누적이 690억 → 1,500억으로 2배 이상 커짐. 규약·신탁계약에서 정의 확정이 선결 과제.

## 문제와 해결

- **이 컨테이너의 LibreOffice가 고장** (`--convert-to`가 txt조차 "source file could not be loaded", 샌드박스 해제해도 동일) → 페이지 수 검증은 **크로미움 headless print-to-pdf + pypdf**로 대체. `/opt/pw-browsers/chromium --headless --no-pdf-header-footer --print-to-pdf=out.pdf file://…` 후 `pypdf`로 쪽수·쪽별 첫/끝 줄 확인.
- **pypdf가 `_cffi_backend` 없음으로 크래시** → `pip install cffi`로 해결 (pypdf의 crypt provider가 cryptography를 임포트하는 경로 문제).
- **쪽 넘침 디버깅**: pypdf로 각 쪽의 첫/마지막 텍스트 줄을 뽑아 어느 `.page` div가 넘쳤는지 특정 → 해당 쪽만 압축하거나 전역 폰트/여백 0.2pt·0.3mm 단위로 조임. 5쪽판은 2회 반복으로 수렴.

## 스킬

### 1. 원격 컨테이너에서 한국어 A4 인쇄물은 "HTML(.page div) → 크로미움 print-to-pdf → pypdf 쪽수 검증" 루프
- `.page { width:210mm; min-height:297mm }` + `@media print { .page { page-break-after:always } }` 구조로 쪽 단위를 명시적으로 설계.
- 크로미움: `/opt/pw-browsers/chromium --headless --disable-gpu --no-sandbox --no-pdf-header-footer --print-to-pdf=out.pdf "file://$PWD/doc.html"`.
- 검증: pypdf로 쪽수 + 쪽별 첫/마지막 줄을 출력해 의도한 경계에서 나뉘는지 확인. 넘치면 폰트 0.2pt·패딩 0.3mm 단위로 전역 압축이 국소 삭제보다 안전.
- LibreOffice는 이 환경에서 신뢰 불가 — docx 쪽수 검증이 필요하면 HTML 경로로 우회.

### 2. 추정 문서는 "기준지표 총괄표 + 산식 인라인 공개"가 신뢰의 핵심
- 모든 계수·단가를 문서 앞에 표(적용값/도출 근거/검증 원자료 3열)로 모으고, 본문 수치마다 산식을 병기하면 제3자가 재현·민감도 분석 가능.
- 연쇄 추정(투자→효과→시장→환류)은 물량 축(GWh)과 돈 축(억원)의 정합을 램프업 가정(신규 개소 첫해 절반 가동)까지 포함해 파이썬 한 스크립트로 일괄 검산해 두면 후속 수정이 안전.

## 레슨

- **"10% 환류"류의 비율 지표는 분모 정의(매출 vs 순이익)를 반드시 먼저 확인**할 것 — 이번 건은 2배 이상 차이가 났고, 사용자가 중간에 순이익 기준으로 정정했다. 추정 착수 전에 물었어야 할 질문.
- 업로드 PDF가 "AI 추정치 기반"임을 명시한 문서일 때도, 파생 추정물에는 같은 급의 일러두기·검증 원자료 목록을 반복 명기해야 인용 사고를 막는다.
