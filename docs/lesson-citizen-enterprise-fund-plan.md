# 레슨 — 시민기업펀드 계획서 작업 (2026-07-22~24)

## 1. 참조 원문이 없을 땐 물어라, 그러나 멈추진 마라

- "시민기금·시민구단 계획서처럼" 요청받았는데 저장소엔 원문이 없었다. 처음엔 완도 핸드오버의 생태계 맥락(시민기금·기본소득 적립·공동체 자산화)으로 최선의 초안을 썼고, 이후 사용자가 **PDF 원문을 제공**하자 서식·법리를 전면 재정렬했다.
- **교훈:** 원문 부재를 이유로 손 놓지 말고 맥락 기반 초안을 내되, 원문이 오면 **서식·용어·번호·법리까지 그 문서에 정렬**시킨다. 시민기금 PDF에서 3층 구조·유사수신·후순위(first-loss)·【사례】 박스·보수/기준/적극 시나리오 표 같은 서식 골격을 그대로 가져와 정합성이 크게 올라갔다.

## 2. 층위가 다른 용어를 '중복'으로 오해하지 말 것

- 나는 '공동체이익회사'와 '시민기업'이 문서에 함께 나오는 것을 **혼재(불일치)**로 보고 통일을 제안했는데, 사용자가 바로잡았다: 둘은 **층위가 다른 별개 개념**이다.
  - **공동체이익회사(CIC)** = 현장의 개별 공동체 기업.
  - **시민기업** = 사회연대경제조직의 한 유형(공동체이익회사의 연합조직이거나, 그들에게 사업·서비스를 제공하며 시민 투자로 운영되는 기업). 펀드의 투자 대상.
- **교훈:** 같은 어근('공동체'/'시민')이라고 통합하려 들지 말 것. 용어가 반복되면 **먼저 관계(포함/상위/서비스)를 확인**하고, 필요하면 정의를 문서에 명시해 구분을 고정한다.

## 3. 내가 만든 hwpx가 실제로 열렸는지는 '재업로드 흔적'으로 확인된다

- 템플릿 없이 만든 hwpx를 전달했고 "한글 열림 미검증"이라 밝혔다. 사용자가 편집 후 재업로드한 파일에 **`Preview/PrvImage.png`·`META-INF/container.rdf`**가 추가돼 있었다 — 이는 한글이 열고 다시 저장했다는 증거다.
- **교훈:** 생성 hwpx의 검증 불가 상황에서, 사용자 재저장본의 엔트리 변화(썸네일·rdf 생성)로 **열림 성공을 사후 확인**할 수 있다. → `skill-hwpx-generate-from-scratch.md`로 승격.

## 4. PDF 추출: 이 환경은 pymupdf

- poppler(pdftotext) 없음, pypdf는 cryptography 바인딩(`_cffi_backend`) 깨져 import 실패. **`pip install pymupdf`(import fitz)**가 확실했다. Read 툴의 PDF 렌더도 pdftoppm 부재로 실패하니, 텍스트만 필요하면 fitz로 직접 뽑는다.

## 5. 번호 중복은 조용히 고치되 사용자에게 알린다

- 한글 편집본에서 '위상'과 '추진배경'이 둘 다 "1"이었다. '위상'을 **번호 없는 도입절**로 두어 본문 1~9장 번호와 내부 참조("3장")를 보존했고, 대안(위상을 정식 1장으로 승격)을 사용자에게 제시했다.
- **교훈:** 문서 번호 같은 형식 결함은 **내부 상호참조를 깨지 않는 방향**으로 최소 수정하고, 판단이 갈리는 지점은 대안과 함께 사용자에게 넘긴다.
