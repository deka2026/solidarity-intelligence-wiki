# 스킬: Supabase Auth 연동 시 주의사항

**작성일:** 2026-06-29

## 배운 것

### 1. 트리거와 signUp 충돌
- `auth.users`에 AFTER INSERT 트리거를 걸면, `sb.auth.signUp()` 호출 시 트리거 내부 에러가 전체 트랜잭션을 롤백시킴
- 에러 메시지: `Database error saving new user` (500)
- 트리거 함수에 `ON CONFLICT DO NOTHING` 추가해도 다른 원인(FK 제약 등)으로 실패 가능
- 해결: 트리거 대신 클라이언트 코드에서 profiles INSERT 처리 권장

### 2. 무료 플랜 이메일 제한
- Supabase 무료 플랜: 이메일 전송 2회/시간 제한
- "Confirm email" 비활성화 토글이 최신 UI에서 위치 변경됨 (Sign In / Providers > Email 안에 없음)
- 이메일 인증 우회 방법: Supabase 대시보드 > Add user > Auto Confirm으로 수동 생성

### 3. GENERATED ALWAYS vs BY DEFAULT
- `GENERATED ALWAYS AS IDENTITY`: 외부 ID 삽입 불가 → JS에서 생성한 ID와 충돌
- `GENERATED BY DEFAULT AS IDENTITY`로 변경하면 외부 ID도 허용
- `ALTER TABLE 테이블명 ALTER COLUMN id SET GENERATED BY DEFAULT;`

### 4. RLS 정책 순서
- 테이블 생성 후 RLS 활성화하면 기본적으로 모든 접근 차단
- 정책을 추가하기 전까지 anon key로는 읽기/쓰기 모두 불가
- 개발 초기에는 `FOR ALL USING(true) WITH CHECK(true)` 로 열어두고 나중에 조임

### 5. 한글 경로와 개발 서버
- `npx serve`는 한글이 포함된 경로에서 실패
- junction link로 우회: `mklink /J "C:\Users\Admin\academy-site-link" "한글경로"`
